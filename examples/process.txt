  Step 1: ingest_resource (io)

  ---
  Step 2: preprocess_multimodal (llm)

  _memorize_preprocess_multimodal()
  └── _preprocess_resource_url(local_path, text, modality, llm_client)
      ├── 查找 preprocess prompt template (PREPROCESS_PROMPTS[modality])
      │
      ├── [document 分支]
      │   └── _preprocess_document(text, template)
      │       ├── prompt = template.format(document_text=text)
      │       ├── llm_client.chat(prompt)  → LLM 返回 XML
      │       └── _parse_multimodal_response(raw, "processed_content", "caption")
      │           → (condensed_text, caption)                                                                                                                                         
      │
      └── [image 分支]                                                                                                                                                                
          └── _preprocess_image(local_path, template)
              ├── llm_client.vision(prompt=template, image_path=local_path)
              │   → Vision API 分析图片
              └── _parse_multimodal_response(raw, "detailed_description", "caption")
                  → (description_text, caption)

  state["preprocessed_resources"] = [{"text": ..., "caption": ...}]

  ---
  Step 3: extract_items (llm)

  _memorize_extract_items()
  └── 对每个 preprocessed_resource:
      └── _generate_structured_entries(resource_url, modality, memory_types, text, categories_prompt_str)
          └── _generate_text_entries()                                                                                                                                                
              └── _generate_entries_from_text()
                  ├── 对每个 memory_type (knowledge/profile/behavior/event/skill 等)                                                                                                  
                  │   └── _build_memory_type_prompt(memory_type, resource_text, categories_str)                                                                                       
                  │       → 填充 MEMORY_TYPE_PROMPTS[mtype] 模板
                  │                                                                                                                                                                   
                  ├── asyncio.gather(*[llm_client.chat(prompt) for prompt in prompts])
                  │   → 并发调用 LLM，每种 memory_type 一次
                  │
                  └── _parse_structured_entries(memory_types, responses)
                      └── 对每个响应:
                          └── _parse_memory_type_response_xml(raw)
                              → 解析 XML: <knowledge><memory><content>...</content>
                                                                <categories><category>...</category>
                              → list[{content, categories}]
                          → entries: [(MemoryType, "content text", ["category_name", ...])]

  state["resource_plans"] = [{resource_url, text, caption, entries: [...]}]

  Step 4: dedupe_merge

  ---
  Step 5: categorize_items (db + vector)

  _memorize_categorize_items()
  └── 对每个 resource_plan:
      ├── _create_resource_with_caption()
      │   ├── embed_client.embed([caption])  → caption_embedding
      │   └── store.resource_repo.create_resource(url, modality, local_path, caption, embedding)
      │
      └── _persist_memory_items(resource_id, entries, ctx, store, embed_client)
          ├── embed_client.embed([content for _, content, _ in entries])                                                                                                              
          │   → 批量生成所有 memory item 的向量
          │                                                                                                                                                                           
          └── 对每个 (memory_type, summary_text, cat_names):

  state += {resources, items, relations, category_updates}

  ---
  四、跨资源累积效果

  3 个资源依次处理，每次 memorize() 调用后：
  - categories_ready=True，跳过初始化
  - 新 items 追加到同一批 category 下
  - Step 6 每次都会用 LLM 增量更新 category summary（原有摘要 + 新 items → 新摘要）

  最终 result["categories"] 返回的是所有资源处理完后的累积摘要状态。

  ---
  五、数据流总结

  文件
   → [ingest] raw_text / local_path                                                                                                                                                   
   → [preprocess] LLM/Vision → condensed text + caption
   → [extract] LLM × N(memory_types) → (type, content, [category_names])                                                                                                              
   → [categorize] embed → MemoryItem + CategoryItem (关联关系)
   → [persist] LLM → 更新 MemoryCategory.summary
   → [response] 序列化返回
