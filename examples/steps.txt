---                                                                                                                                                                                 
  example_video_batch.py 全流程拆解                                                                                                                                                   
                                                                                           
  一、顶层流程                                                                                                                                                                        
                                                                                                                                                                                      
  main(video_dir, concurrency, output_dir)
  ├── 1. discover_videos()          → list[VideoTask]                                                                                                                                 
  ├── 2. build_unique_categories()  → list[dict]            
  ├── 3. MemoryService.__init__()   → service
  ├── 4. run_batch()                → list[TaskResult], list[dict(category)]
  ├── 5. 统计 BatchStats
  ├── 6. write_category_md()        → 每个 category 一个 .md

  二、启动阶段

  2.1 discover_videos()

  video_dir.glob("**/*.mp4")  →  按文件名排序
  └── 对每个 mp4:
      parse_category_from_filename(mp4)
      ├── mp4.stem = "interview_01.cooking"
      ├── rsplit(".", 1) → ["interview_01", "cooking"]

  → [{"name": "cooking", "description": "..."}, {"name": "travel", ...}, ...]

  2.3 MemoryService.init()

  MemoryService(llm_profiles={...}, memorize_config={"memory_categories": unique_categories})
  ├── 验证 llm_profiles → LLMProfilesConfig
  │   └── 自动补充 "embedding" profile = "default" profile
  ├── 构建 category_configs (N个去重后的分类)

  ---                                                                                                                                                                                 
  三、并发执行阶段                                          
                                                                                                                                                                                      
  3.1 run_batch()
                                                                                                                                                                                      
  run_batch(tasks, service, concurrency=4)                  
  ├── semaphore = asyncio.Semaphore(4)
  ├── coros = [process_one(service, task, semaphore) for task in tasks]
  │   → 100个协程对象，此时尚未执行
  └── asyncio.gather(*coros)                                                                                                                                                          
      → 同时提交所有协程到事件循环                                                                                                                                                    
      → Semaphore 保证任意时刻最多 4 个并发执行                                                                                                                                       
      → 所有完成后返回 list[TaskResult]                                                                                                                                               
                                                                                                                                                                                      
  3.2 process_one() — 单个视频的执行                        

  async with semaphore:          ← 获取令牌，超出 concurrency 则等待
      service.memorize(resource_url=str(path), modality="video")
      → 返回 TaskResult

  四、service.memorize() 内部详细流程（video 模态）

  阶段 0：分类初始化（仅第一个拿到锁的协程执行，其余等待）

  memorize()
  └── _ensure_categories_ready(ctx, store, user_scope)
      └── ctx.categories_ready == False → _initialize_categories()
          ├── 对每个 CategoryConfig 生成 embedding text
          │   "cooking: Videos related to cooking"
          │   "travel: Videos related to travel"
          │   ...                                                                                                                                                                     
          ├── llm_client("embedding").embed([N个文本])
          │   → N个向量 (并发批量请求)                                                                                                                                                
          └── 对每个 (cfg, vec):                            
              store.memory_category_repo.get_or_create_category(name, description, embedding)
              ctx.category_ids.append(cat.id)
              ctx.category_name_to_id["cooking"] = cat_id_1
              ...
          → ctx.categories_ready = True

  ▎ 并发安全说明：_ensure_categories_ready 检查 ctx.categories_ready，但多个协程可能同时通过检查。由于 asyncio 是单线程事件循环，await 点才会切换，_initialize_categories 内部的 await
   embed() 之前不会被抢占，所以实际上只有第一个协程会真正执行初始化，后续协程进入时 categories_ready 已为 True。

  Step 2: preprocess_multimodal (llm)

  _memorize_preprocess_multimodal()
  └── _preprocess_resource_url(local_path, text=None, modality="video", llm_client)
      ├── 查找 template = PREPROCESS_PROMPTS["video"]
      │   → 视频分析的 prompt 模板
      ├── template 存在，且 modality != "audio"，且不需要 text

          └── _preprocess_video(local_path, template, llm_client)
              ├── VideoFrameExtractor.is_ffmpeg_available()  → 检查 ffmpeg                                                                                                            
              ├── VideoFrameExtractor.extract_middle_frame(local_path)
              │   → 用 ffmpeg 提取视频中间帧                                                                                                                                          
              │   → frame_path = "/tmp/xxxxx.jpg"

  ---
  Step 3: extract_items (llm)

  _memorize_extract_items()
  └── 对 preprocessed_resources[0] (video 只有1个):
      res_url = "path/to/video.cooking.mp4"  (total_segments=1，不加 segment 后缀)
      text = "A person cooking pasta in a kitchen..."
      caption = "Person cooking pasta"

      _generate_structured_entries(res_url, "video", memory_types, text, categories_prompt_str)                                                                                       
      └── _generate_text_entries(text, ...)
          └── _generate_entries_from_text(text, memory_types, categories_prompt_str)                                                                                                  
              ├── 对每个 memory_type (knowledge/profile/behavior/event/skill):
              │   _build_memory_type_prompt(mtype, resource_text=description, categories_str)
              │   → 填充模板，例如 knowledge 类型:
              │     "从以下内容提取知识类记忆...
              │      内容: A person cooking pasta...
              │      可用分类: - cooking: Videos related to cooking
              │                - travel: ..."
              │
              ├── asyncio.gather(*[llm_client.chat(prompt) for prompt in prompts])
              │   → 并发调用 LLM，每种 memory_type 一次 (5个并发请求)

  category_updates 写入 inmemory store → 无锁冲突（asyncio 单线程）
  category summary 更新 → 每次 memorize 结束时独立更新，后写覆盖前写
                                                                                                                                                                                      
  ---
  六、数据流总结                                                                                                                                                                      
                                                            
  N个 mp4 文件
   → [discover] 文件名解析 → VideoTask(path, category_name)
   → [去重] → M个唯一 category → MemoryService 注册 + embed
   → [并发 × concurrency]
       每个文件:
         [ingest]      → local_path (video 无 raw_text)
         [preprocess]  → ffmpeg 抽帧 → Vision API → description + caption
         [extract]     → LLM × N(memory_types) 并发 → (type, content, [cat_names])
         [categorize]  → embed items → 写入 inmemory store → category_updates
         [persist]     → LLM 更新 category summary（增量累积）
         [response]    → TaskResult(items_count, elapsed)
   → [汇总] → category .md 文件 + results.json

  关键并发特性：
  - asyncio.Semaphore 控制最大并发数，防止 API 限流
  - asyncio.gather 内部每个 memorize 调用的 Step 3（extract_items）自身也会并发调用多个 memory_type 的 LLM
  - inmemory store 无需加锁（asyncio 单线程，await 点之间不会被抢占）
  - category summary 在高并发下存在后写覆盖问题：多个协程同时更新同一 category 时，最后完成的那个会覆盖之前的摘要
